# ギブアップは敗北として記録し、答えを見る唯一の正規ルートとする

- Date: 2026-07-04
- Type: ADR
- Status: Active
- Author: masahiro.kasatani

## Context

プレイ中に手が止まった時、想定コマンドと解説をノーコストで見られる導線があると、「考えずに答えを見る」が最短ルートになり、CLAUDE.md の「暗記クイズにしないこと」という方針と衝突する。一方で、詰まったまま先に進めなくなる体験も避けたい。答えを見る手段は用意しつつ、必ず何らかのコストを伴わせる。

## Decision

- **「回答を見る」= ラウンドを敗北として即終了し、既存のリザルト画面（想定コマンド + 分解解説）に遷移する。** 専用の解説 UI は作らない。時間切れ（`timeout`）と同じ失敗結果画面を再利用し、見出し文言だけを変える（`result.timeout` vs `result.gaveUp`）
- **ギブアップは `Attempt.success = false` で記録し、`revealed: true` を追加する。** スキップも同様に `skipped: true` を追加する。どちらも省略可能なフィールドで、既存の保存データとの後方互換のためスキーマバージョンは上げない（欠落 = false）。将来の弱点分析で「時間切れ」「自発的ギブアップ」「スキップ（未挑戦）」を区別できるようにする
- **ギブアップとスキップは明確に別機能として共存させる。** スキップ（`s`）は解説なしで即座に次の問題へ進む「この問題は今は解かない」という選択で、リザルト画面を経由しない。ギブアップ（`?`）は「答えが知りたい」という選択で、必ずリザルト画面と解説を経由する。両者ともキーボード操作の「プレイ中キーヒント」に並べて表示する（`[s] skip` / `[?] give up`）
- **発動条件の出し惜しみをしない。** 残り時間や試行回数によるロックは設けず、プレイ中いつでも発動できる。コストは「失敗記録として残る」「streak がリセットされる」ことで十分とする
- **`?` と `s` はどちらも Normal mode かつ `inputState.phase === "idle"`（オペレータ/f・t/テキストオブジェクトが pending でない）ときだけ meta キーとして扱う。** 両方とも Vim の `f`/`t` モーションの対象文字になり得る文字（例: `f?` で次の `?` を検索）であるため、これを idle 限定にしないと、モーション入力の途中で意図せずギブアップ/スキップが発火しうる。Insert mode では常にこのチェック自体を行わず、通常の置換テキストとして入力される

## Consequences

- `RoundVerdict`（`src/core/score.ts`）に `"gaveUp"` を追加し、`nextStreak` は `timeout` と同様にリセットする
- `ChallengeRound` の `endRound`（`src/app/pages/LevelRound.tsx`）は `success` の判定を `verdict !== "timeout"` から `verdict === "great" || verdict === "verbose"` という明示的な包含チェックに変更した。以後 `RoundResult` に新しい失敗系 verdict を足すたびに、ここを触らなくても自動的に失敗扱いになる
- スキップは既存の「即座に次へ」という体験を変えずに、記録だけを追加した（`recordAttempt` を直接呼び、`endRound` は経由しない - リザルト画面を出さないため）
- `?` を「idle 限定の meta キー」として扱う設計は、副次的に既存の `s`（skip）にも同じ制約を後付けで適用する形になった。以前は `s` がどんな Normal mode 状態でも即座にスキップとして解釈されており、`f?`/`fs` のような（現状データには存在しない）お題を将来追加すると壊れる潜在バグだったため、このタイミングで合わせて修正した

## Alternatives Considered

- **専用の「解説モーダル」を作り、プレイを継続しながら答えを覗ける導線にする** — 実装コストが増えるだけでなく、「ノーコストで答えを見られる」状態そのものが望ましくないため却下
- **ギブアップに残り時間などの発動条件を課す（例: 残り5秒未満のみ可）** — 出し惜しみによってプレイヤーを苛立たせるだけで、記録によるコスト（敗北扱い・streak リセット）で十分だと判断し却下
